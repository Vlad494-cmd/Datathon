----------------------------------------------------------
1. CHALLENGE OVERVIEW
----------------------------------------------------------
- Goal: Design a data-driven EV charging network for Spain's interurban road system.
- Primary task: Propose optimal locations for public-use charging points on interurban routes, minimizing the number of stations while adequately covering EV mobility demand.
- Secondary task: Identify "friction points" — locations where grid capacity constraints make EV charging deployment unfeasible or risky.
- Target horizon: 2027 operational scenario (all models and projections must target this year).
- This is NOT a pure coding challenge. Research, strategy, data sourcing, and business acumen are equally valued.

----------------------------------------------------------
2. SCOPE OF ANALYSIS
----------------------------------------------------------
- Geography: Interurban transport network in Spain ONLY.
- Eligible road types: autopistas, autovías, carreteras nacionales (as classified by Ministry of Transport dataset).
- EXCLUDED: Urban road sections (regardless of municipality size). Large urban centers must be excluded from the primary focus.
- Rationale: Focus on long-distance travel and "range anxiety" — the key bottleneck to mass EV adoption.
- Model must incorporate BOTH transport geography AND electrical grid capacity. A geographically ideal location is invalid if the local substation cannot support high-power chargers.

----------------------------------------------------------
3. THREE CORE OBJECTIVES
----------------------------------------------------------

OBJECTIVE 1 — Charging Network Optimization (PRIMARY)
- Propose the minimum number of charging stations needed to adequately cover interurban EV mobility demand.
- "Optimal" = maximizing utilization and driver convenience while minimizing capital expenditure.
- Variables to consider: average EV autonomy, seasonal traffic fluctuations, major transit corridors, spacing to prevent drivers from being stranded.
- The network must be economically sparse yet functionally seamless across Spain.
- Required outputs: Global Network KPIs (File 1) + Proposed Charging Locations (File 2).

OBJECTIVE 2 — Grid Viability Analysis (SECONDARY)
- Identify areas where electrical grid congestion limits or prevents deployment.
- Cross mobility data with grid capacity maps from distributors: i-DE (Iberdrola), Endesa, Viesgo.
- Find "friction points": locations where high charging demand meets low grid hosting capacity.
- Required output: Geospatial friction point log (File 3).

OBJECTIVE 3 — Strategy and Value Proposition
- Translate quantitative analysis into an actionable strategic roadmap for Iberdrola.
- Address trade-offs between user demand, infrastructure costs, and grid limitations.
- Propose phased deployment approach aligned with EV adoption growth.
- Required output: Integrated into the Analytical Report and Final Pitch.

6. HANDLING DATA GAPS & ASSUMPTIONS
----------------------------------------------------------
- Incomplete or imperfect data is expected. Teams are encouraged to make informed assumptions.
- All assumptions MUST be documented in the Analytical Report with the following structure:
    1. Document Your Logic: Clearly state proxy values or assumptions (e.g., grid capacity classification thresholds).
    2. Show Your Research: Back assumptions with industry standards, external sources, or technical reports.
    3. The Expert Perspective: Justify why the assumption makes sense in the context of real infrastructure planning.
- Evaluation philosophy: Robust, well-justified reasoning under data scarcity is rewarded MORE than a lucky correct number.

--- DELIVERABLE 2: OUTPUT DATASETS (3 CSV FILES) ---

** FILE 1: Global Network KPIs (Summary Scorecard) **
File name: File 1.csv
A SINGLE summary row. Fields:

  Field                            | Data Type | How Obtained
  ---------------------------------|-----------|------------------------------------------------------
  total_proposed_stations          | Integer   | Row count of File 2.
  total_existing_stations_baseline | Integer   | Count of interurban stations from National Access Point dataset (autopistas, autovías, carreteras nacionales only).
  total_friction_points            | Integer   | Row count of File 3.
  total_ev_projected_2027          | Integer   | Direct output of mandatory datos.gob.es GitHub fork (projected total EV fleet in Spain for 2027).

** FILE 2: Proposed Charging Locations (Objective 1) **
File name: File 2.csv
One row per proposed new charging station. Fields:

  Field               | Data Type   | Description
  --------------------|-------------|-------------------------------------------------------------
  location_id         | String      | Unique sequential ID (e.g., IBE_001, IBE_002).
  latitude            | Float       | Decimal degrees, WGS84.
  longitude           | Float       | Decimal degrees, WGS84.
  route_segment       | String      | Official road designation per Ministry of Transport (e.g., A-3, AP-7, N-2).
  n_chargers_proposed | Integer     | Number of charging points at this site. Must be justified in Analytical Report based on projected local demand.
  grid_status         | Categorical | Grid capacity classification based on distributor data. Accepted values: Sufficient | Moderate | Congested. Teams must define and document their own classification thresholds in the Analytical Report.

** FILE 3: Friction Points (Objective 2) **
File name: File 3.csv
Only locations from File 2 where grid_status = Moderate OR Congested. Locations with grid_status = Sufficient MUST NOT appear here. Fields:

  Field                | Data Type   | Description
  ---------------------|-------------|-------------------------------------------------------------
  bottleneck_id        | String      | Unique sequential ID (e.g., FRIC_001, FRIC_002).
  latitude             | Float       | Decimal degrees, WGS84.
  longitude            | Float       | Decimal degrees, WGS84.
  route_segment        | String      | Official road designation per Ministry of Transport (e.g., A-3, AP-7, N-2).
  distributor_network  | String      | Accepted values: i-DE | Endesa | Viesgo.
  estimated_demand_kw  | Float       | = n_chargers_proposed × 150 kW. (150 kW is FIXED for ALL teams. No other value accepted.)
  grid_status          | Categorical | Only Moderate or Congested. Sufficient is invalid here.

** MANDATORY STANDARDIZATION RULES: **
  - Rule 1: grid_status MUST be assigned based on the "Capacidad disponible / Available capacity (MW)" field from the corresponding distributor dataset (i-DE, Endesa, or Viesgo). Teams must identify the nearest substation to each proposed location, define their own classification thresholds, and document BOTH the spatial matching methodology AND the threshold justification in the Analytical Report.
  - Rule 2: estimated_demand_kw = n_chargers_proposed × 150 kW. This is fixed and non-negotiable.
  - Rule 3: File 3 only contains Moderate or Congested locations from File 2. Sufficient locations are excluded.
